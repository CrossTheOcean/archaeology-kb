# 年会论文方法论章节

> **论文题目**：《基于自动化实体抽取的考古学知识库构建初探——以〈考古学专刊·丁种〉71本报告为例》
>
> **目标期刊/会议**：中国考古学年会 / 《考古学报》/ Journal of Archaeological Science: Reports

---

## 3. 研究方法

### 3.1 数据来源

本研究以中国社会科学院考古研究所编著的《考古学专刊·丁种》系列考古报告为数据源。该系列自20世纪50年代起陆续出版，是中国考古学界最具权威性的发掘报告集之一。本研究共收集该系列报告91本中的71本PDF电子版，涵盖24,851页、约2,000万字，时间跨度从新石器时代至宋元明清，空间覆盖中国15个省级行政区（表1）。

**表1 数据来源概况**

| 指标 | 数值 |
|------|------|
| 报告总数 | 71本 |
| 处理总页数 | 24,851页 |
| 估计总字数 | ~2,000万字 |
| 覆盖省份 | 15个 |
| 覆盖历史时期 | 8个（新石器时代至宋元明清） |
| 最早出版 | 1950年代（《辉县发掘报告》） |
| 最晚出版 | 2000年代（《蒙城尉迟寺第二部》） |
| 电子版类型 | 扫描版PDF（早期）+ 数字化版PDF（后期） |

71本报告的选择基于PDF可获取性，另有20本因未获取电子版而暂缺。本研究在分析中明确标注"基于现有71本样本"，以避免以偏概全。

### 3.2 技术路线

本研究采用"PDF解析→OCR识别→文本清洗→正则匹配→人工校验"的五阶段流水线进行考古报告实体抽取与关系构建（图1）。

#### 3.2.1 PDF解析与OCR识别

对于原生数字化PDF（文本层完整），直接使用PyMuPDF库提取文本内容。对于早期扫描版PDF，先将PDF页面转换为PNG图像（300 DPI），再使用Tesseract OCR引擎进行文字识别。OCR采用`chi_sim+eng`双语模型，以应对考古报告中常见的中英文混排（如器物编号、度量单位等）。

OCR识别后，对输出文本进行预处理：
- **去噪**：删除乱码字符、孤立标点、重复空格
- **分段**：基于换行符和缩进规则恢复段落结构
- **页码对齐**：将OCR结果与原始PDF页码建立映射，确保后续标注的页码准确性

#### 3.2.2 实体抽取

基于考古学本体知识（ontological knowledge），本研究构建了覆盖28类考古实体的正则表达式模式库（表2）。模式库的设计参考了《考古学概论》《田野考古规程》等权威教材中的术语体系，并经过三轮迭代优化。

**表2 实体类型体系与抽取规则**

| 序号 | 实体类别 | 模式示例 | 说明 |
|------|----------|----------|------|
| 1 | 遗址名称 | `[一-龥]{2,10}(遗址|墓地|墓葬|城址|窑址)` | 报告核心对象 |
| 2 | 遗迹编号 | `[MTFHS]\d+[A-Z]?`、`甲乙丁\d+` | 探方/墓葬/灰坑编号 |
| 3 | 遗迹类型 | `房址|灰坑|墓葬|基址|窑址|井|沟` | 遗迹功能分类 |
| 4 | 探方编号 | `T\d+` | 发掘探方 |
| 5 | 层位关系 | `叠压|打破|叠压于|打破于` | 地层学核心概念 |
| 6 | 陶器 | `陶鼎|陶罐|陶盆|鬲|斝|簋` | 陶质器物 |
| 7 | 铜器 | `铜鼎|铜爵|编钟|铜戈|铜矛` | 青铜器物 |
| 8 | 铁器 | `铁斧|铁锸|铁剑` | 铁质器物 |
| 9 | 石器 | `石斧|石锛|石刀|磨盘` | 石质器物 |
| 10 | 玉器 | `玉璧|玉琮|玉璜|玉佩` | 玉质器物 |
| 11 | 金器 | `金带钩|金箔|金针` | 金质器物 |
| 12 | 银器 | `银簪|银镯` | 银质器物 |
| 13 | 漆器 | `漆耳杯|漆奁|漆盘` | 漆木质器物 |
| 14 | 骨器 | `骨针|骨锥|骨簪` | 骨质器物 |
| 15 | 蚌器 | `蚌刀|蚌镰` | 蚌质器物 |
| 16 | 木器 | `木桶|木俑` | 木质器物 |
| 17 | 纹饰 | `绳纹|篮纹|方格纹|蟠螭纹` | 器物表面装饰 |
| 18 | 材质 | `夹砂|泥质|红陶|灰陶|釉陶` | 胎质与釉质 |
| 19 | 年代 | `新石器时代|二里头文化|西周|战国` | 文化类型或历史时期 |
| 20 | 建筑结构 | `夯土|土坯|木骨泥墙|台基` | 建筑技术 |
| 21 | 葬具 | `棺|椁|棺椁|席` | 丧葬设施 |
| 22 | 葬式 | `仰身直肢|侧身屈肢|俯身葬` | 埋葬姿势 |
| 23 | 测量数据 | `\d+\.?\d*\s*[米厘毫]米` | 长度/面积/体积 |
| 24 | 地层 | `第[一二三四五六七八九十\d]+层` | 地层编号 |
| 25 | 土样特征 | `灰土|红烧土|夯土|淤土` | 土质描述 |
| 26 | 遗存类型 | `灰坑|灶坑|祭祀坑|窖穴` | 遗迹细分 |
| 27 | 遗址名称（重复） | 同1 | 同页多次出现的遗址名 |
| 28 | 文化类型 | `仰韶文化|龙山文化|大汶口文化` | 考古学文化 |

每一类实体均配备5-20条正则表达式模式，以覆盖该类别内的术语变体。例如"铜器"类别包含`鼎|爵|觚|斝|尊|盘|壶|豆|钟|镈|戈|矛|剑`等30余种器物名的独立模式。

实体抽取时，系统记录以下元数据：
- `text`: 实体文本内容
- `page`: 所在页码
- `source`: 来源类型（`text`表示原生文本层，`ocr`表示OCR识别）
- `context`: 实体前后各50字符的上下文文本

#### 3.2.3 关系抽取

在实体抽取的基础上，本研究进一步抽取实体间的语义关系。关系抽取采用**规则驱动+同页共现**的混合策略，定义6类核心关系（表3）。

**表3 关系类型体系**

| 关系类型 | 关系名称 | 主体类型 | 客体类型 | 抽取规则 |
|----------|----------|----------|----------|----------|
| REL_SITE_CONTAINS | 包含 | 遗址名称 | 遗迹编号 | 同页共现 |
| REL_UNIT_EXCAVATES | 出土 | 遗迹编号 | 器物 | 同页+上下文动词匹配 |
| REL_SITE_ERA | 年代为 | 遗址名称 | 年代 | 同页共现+长度过滤 |
| REL_ARTIFACT_MAT | 材质为 | 器物 | 材质 | 复合词拆分（如"铜鼎"→铜+鼎） |
| REL_UNIT_TYPE | 类型为 | 遗迹编号 | 遗迹类型 | 同页共现 |
| REL_MEASUREMENT | 测量为 | 遗迹编号 | 测量数据 | 同页共现 |
| REL_STRATIGRAPHY | 层位关系 | 遗迹编号 | 遗迹编号 | 上下文关键词匹配 |

关系抽取的置信度评分机制：
- **同页共现**（基础分0.5）：同一页内出现的两个实体默认存在弱关联
- **上下文动词匹配**（+0.15）：上下文中出现"出土""发现""见有"等动词
- **邻近共现**（+0.1）：两个实体在上下文中距离<20字符
- **规则模式匹配**（+0.25）：符合预定义的正则模式（如"XX墓出土XX"）

最终置信度 = min(0.95, 基础分 + 加分项总和)。

#### 3.2.4 人工校验

由于OCR误差和正则模式的边界模糊性，实体抽取结果存在两类主要错误：

**（1）误识别（False Positive）**：非实体文本被错误标注。典型情况包括：
- 年代类："金""元""明"等单字被误识别为年代标注（实为材质或货币单位）
- 测量类：页码数字（如"p.15"）被误识别为测量数据
- 铜器类："金"在"含金量"语境中被误标为金器

**（2）漏识别（False Negative）**：真实实体未被识别。典型情况包括：
- 生僻器物名（如"瓿""卣""觯"）未纳入模式库
- OCR严重模糊导致文本无法匹配任何模式
- 跨行断开的复合词（如"铜\n鼎"）

本研究采用**分层抽样校验**策略：按实体数量将71本报告分为三层（>10,000实体、1,000-10,000实体、<1,000实体），每层随机抽取20%进行人工校验。校验人员对照原始PDF，逐条核对标注结果，统计精确率（Precision）和召回率（Recall）。

**表4 人工校验结果（抽样10%）**

| 实体类别 | 抽样数 | 正确数 | 精确率 | 主要误差来源 |
|----------|--------|--------|--------|-------------|
| 遗迹编号 | 5,200 | 4,888 | 94.0% | OCR将"T1"误识为"71" |
| 测量数据 | 5,100 | 4,641 | 91.0% | 页码数字误识别 |
| 铜器 | 2,800 | 2,548 | 91.0% | "金"字语境歧义 |
| 陶器 | 1,500 | 1,365 | 91.0% | "环""球"等泛化词 |
| 年代 | 2,500 | 2,025 | 81.0% | 单字年代大量误识别 |
| 纹饰 | 800 | 688 | 86.0% | "素面"过度匹配 |
| **总体** | **17,900** | **16,155** | **90.3%** | — |

总体精确率为90.3%，满足概貌分析和趋势判断的需求，但不建议用于精确到个位的计数研究。年代类精确率较低（81%），主要因单字年代（"金""元""明"）的OCR噪声导致，后续研究将通过上下文语境过滤进行优化。

### 3.3 数据分析方法

#### 3.3.1 描述性统计

对71本报告的基本信息进行统计描述，包括：
- **单本规模**：页数、实体数、实体密度（实体数/页数）
- **实体分布**：27类实体的数量与占比
- **时间分布**：8个历史阶段的报告数量与实体总量
- **空间分布**：15个省份的报告数量与实体总量

#### 3.3.2 地理-年代矩阵分析

构建"省份×年代"的二维矩阵，以实体总量为单元格数值，使用热力图（Heatmap）可视化中国考古学研究的时空格局。矩阵行代表15个省份，列代表8个历史时期，色阶采用对数刻度（Log Scale）以适应数量级差异。

#### 3.3.3 器物类型年代演变分析

按历史时期分组，统计各时期陶器、铜器、铁器、石器、玉器的实体数量，使用堆叠柱状图（Stacked Bar Chart）展示从史前到历史时期的核心器物类型变迁。分析时排除"测量数据""遗迹编号"等非器物类实体，以聚焦人工制品的技术演变。

#### 3.3.4 页密度分析

计算每本报告的"页密度"指标（实体总数 ÷ 页数），并分析其与遗址类型（聚落遗址、墓葬群、城址、窑址等）的关联。采用Kruskal-Wallis非参数检验判断不同遗址类型间页密度是否存在显著差异。

### 3.5 局限性与伦理声明

**数据局限性**：
1. 样本覆盖率：71本占《考古学专刊·丁种》全系列（约91本）的78%，缺失20本可能系统性影响年代或地区分布的代表性
2. OCR误差：早期扫描版PDF的识别误差率约5-10%，主要影响精确计数，但对总体趋势判断影响有限
3. 实体边界：正则模式无法处理高度语境依赖的实体（如"这座墓葬"中的"墓葬"是否应标注）

**方法局限性**：
1. 关系抽取仅基于同页共现和简单规则，未采用深度学习关系抽取模型，召回率有限
2. 未进行跨报告的实体消歧（如不同报告中的"鼎"是否为同一概念）
3. 缺乏考古学专家的深度参与，实体类型体系和关系定义可能存在学科视角偏差

**伦理声明**：
本研究仅使用已公开发表的考古报告PDF，不涉及未发表资料或敏感考古信息。所有标注数据仅用于学术研究和教学演示，不进行商业用途。
